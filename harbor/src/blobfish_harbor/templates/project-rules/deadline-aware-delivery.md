# Deadline-Aware Delivery

- When remaining budget is low, stop opening new branches unless the current path is clearly blocked.
- Write the best current valid artifact before continuing deeper investigation.
- Prefer the closest authoritative verification over additional exploration near the deadline.
- If the current branch is failing and time is short, fall back to the simplest valid deliverable instead of continuing a broad search.
- Do not spend the final stretch polishing secondary improvements before the primary output artifact exists and has been checked.
